# 🚀 RCL-IPtable Pro

**اسکریپت حرفه‌ای تانل و مدیریت و فوروارد پورت‌های TCP و UDP در لینوکس**

مدیریت دستی قوانین `iptables` می‌تواند زمان‌بر و مستعد خطا باشد، به‌ویژه زمانی که سرویس‌هایی مانند Docker، KVM یا Virtualizor نیز روی سرور فعال هستند.
**RCL-IPtable Pro** با ارائه یک رابط کاربری تعاملی، امکان ایجاد، مدیریت و مانیتورینگ قوانین فوروارد پورت را به شکلی ایمن، سریع و پایدار فراهم می‌کند.

---

## ✨ قابلیت‌ها

 🛡️ **ایمنی بالا**

  * جلوگیری از تداخل با Docker، KVM، Virtualizor و سایر قوانین فایروال

 🌐 **فوروارد چندگانه**

  * انتقال پورت به چندین سرور مقصد
  * پشتیبانی از TCP
  * پشتیبانی از UDP
  * پشتیبانی هم‌زمان از TCP و UDP

 🤖 **شناسایی خودکار تنظیمات شبکه**

  * تشخیص خودکار اینترفیس اصلی
  * تشخیص خودکار مقدار MTU

 ⚡ **بهینه‌سازی شبکه**

  * فعال یا غیرفعال کردن BBR
  * فعال یا غیرفعال کردن FQ Queue
  * فعال یا غیرفعال کردن MSS Clamping

 📊 **مانیتورینگ لحظه‌ای**

  * نمایش تعداد بسته‌های عبوری
  * نمایش حجم ترافیک هر مسیر
  * بروزرسانی خودکار آمار

 🩺 **بررسی سلامت مسیرها**

  * بررسی قوانین DNAT
  * بررسی قوانین FORWARD
  * بررسی دسترسی‌پذیری سرور مقصد

 🔄 **حفظ تنظیمات پس از راه‌اندازی مجدد**

  * ذخیره خودکار تغییرات
  * بارگذاری خودکار قوانین پس از ریبوت

 💾 **پشتیبان‌گیری و بازیابی**

  * تهیه نسخه پشتیبان از تنظیمات
  * بازیابی کامل تنظیمات در چند ثانیه

 🎯 **جلوگیری از تداخل پورت‌ها**

  * تشخیص خودکار پورت‌های تکراری
  * پیشنهاد جایگزین مناسب در صورت وجود تداخل

## ⬇️ نصب

دستورات زیر را با کاربر `root` اجرا کنید:

```bash
curl -Ls https://raw.githubusercontent.com/Emmanuel-RCL/iptable-pro/main/iptable-pro.sh -o iptable-pro.sh

chmod +x iptable-pro.sh

bash iptable-pro.sh
```

پس از اولین اجرا، برنامه به صورت خودکار در مسیر زیر نصب می‌شود:

```text
/usr/local/bin/rcl-iptable
```

از این پس برای ورود به پنل مدیریت کافی است دستور زیر را اجرا کنید:

```bash
rcl-iptable
```

## 🎮 نحوه استفاده

پس از اجرای برنامه، یک منوی تعاملی در اختیار شما قرار می‌گیرد که از طریق آن می‌توانید:

* ایجاد مسیر جدید
* ویرایش مسیرهای موجود
* حذف مسیرها
* مشاهده وضعیت و آمار ترافیک
* فعال‌سازی یا غیرفعال‌سازی بهینه‌سازی‌های شبکه
* تهیه نسخه پشتیبان
* بازیابی تنظیمات

را بدون نیاز به وارد کردن دستی دستورات `iptables` انجام دهید.

## ⚠️ نکات مهم

* اجرای برنامه نیازمند دسترسی `root` است.

* زنجیره‌های اصلی سیستم را حذف یا پاک‌سازی نمی‌کند.
* پیش از استفاده در محیط‌های حساس یا عملیاتی، آزمایش در محیط تست توصیه می‌شود.
